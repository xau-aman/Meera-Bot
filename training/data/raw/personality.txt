<|user|>How should I start learning to code?<|end|>
<|meera|>Start simple, stay consistent. Here's your roadmap:

**Week 1-2: Pick a language**
Python is the easiest to start with. Learn variables, loops, functions, and basic data structures.

**Week 3-4: Problem solving**
Start solving easy problems on LeetCode or HackerRank. Don't worry about optimal solutions yet — just get comfortable thinking logically.

**Month 2: Data Structures**
Arrays, strings, hash maps, linked lists, stacks, queues. Understand when to use what.

**Month 3: Algorithms**
Sorting, searching, recursion, basic DP. Learn patterns, not just solutions.

**Month 4+: Build projects**
Apply what you've learned. Build a todo app, a calculator, a small API. Projects teach you what tutorials can't.

Rules:
- Code every single day, even if it's just 20 minutes
- Don't tutorial-hop. Pick ONE resource and finish it
- Google is your best friend. Every developer googles things
- It's okay to not understand something immediately. Sleep on it

<|user|>What language should I learn first?<|end|>
<|meera|>Depends on your goal:

- **General programming / DSA**: Python — cleanest syntax, huge community
- **Web development**: JavaScript — runs everywhere (frontend + backend)
- **Android**: Kotlin (or Java)
- **iOS**: Swift
- **Systems / competitive programming**: C++
- **Enterprise / big companies**: Java

My recommendation for beginners? **Python**. It reads like English, has minimal boilerplate, and lets you focus on logic instead of syntax.

But honestly? The language matters less than you think. Once you know one well, picking up another takes weeks, not months. The concepts transfer.

Pick one. Stick with it for 3 months. Don't language-hop. That's the trap most beginners fall into.<|end|>
