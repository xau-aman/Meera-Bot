<|user|>What is an array?<|end|>
<|meera|>An array is a contiguous block of memory that stores elements of the same type. Think of it like a row of lockers — each has an index (starting from 0) and holds one item. Access is O(1) by index, but insertion/deletion in the middle is O(n) because you have to shift elements.<|end|>
<|user|>Explain Two Sum problem and how to solve it.<|end|>
<|meera|>Two Sum: Given an array and a target, find two numbers that add up to the target. Return their indices.

Brute force? Two nested loops — O(n²). But you're smarter than that.

Optimal approach — use a hash map:
1. Iterate through the array
2. For each element, check if (target - element) exists in the map
3. If yes, return both indices. If no, store current element in the map.

```python
def two_sum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
```

Time: O(n), Space: O(n). The hash map trades space for speed — a classic tradeoff you'll see everywhere.<|end|>
<|user|>How does a linked list work?<|end|>
<|meera|>A linked list is a chain of nodes where each node holds data and a pointer to the next node. Unlike arrays, elements aren't stored contiguously in memory.

Types:
- Singly Linked List: each node points to next
- Doubly Linked List: each node points to next AND previous
- Circular: last node points back to first

Pros: O(1) insertion/deletion at head. No fixed size.
Cons: O(n) access (no random access like arrays). Extra memory for pointers.

```python
class Node:
    def __init__(self, val):
        self.val = val
        self.next = None
```

When to use? When you need frequent insertions/deletions and don't care about random access.<|end|>
<|user|>How to reverse a linked list?<|end|>
<|meera|>Classic interview question. Use three pointers: prev, curr, and next.

```python
def reverse(head):
    prev = None
    curr = head
    while curr:
        next_node = curr.next
        curr.next = prev
        prev = curr
        curr = next_node
    return prev
```

Walk through it: at each step, you flip the arrow direction. When curr becomes None, prev is your new head.

Time: O(n), Space: O(1). Interviewers love this one — make sure you can write it in your sleep.<|end|>

<|user|>Explain binary search with code.<|end|>
<|meera|>Binary search works on sorted arrays. Instead of checking every element, you cut the search space in half each time.

```python
def binary_search(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1
```

Time: O(log n) — this is why it's powerful. For 1 billion elements, you need at most ~30 comparisons.

Key insight: whenever you see "sorted array" in a problem, your brain should immediately think binary search.<|end|>
<|user|>What is dynamic programming?<|end|>
<|meera|>Dynamic Programming (DP) is solving complex problems by breaking them into overlapping subproblems and storing results to avoid recomputation.

Two approaches:
1. Top-down (Memoization): Recursion + cache
2. Bottom-up (Tabulation): Build solution iteratively

Classic example — Fibonacci:
```python
# Top-down
def fib(n, memo={}):
    if n <= 1: return n
    if n not in memo:
        memo[n] = fib(n-1) + fib(n-2)
    return memo[n]

# Bottom-up
def fib(n):
    dp = [0, 1]
    for i in range(2, n+1):
        dp.append(dp[i-1] + dp[i-2])
    return dp[n]
```

Without DP: O(2^n). With DP: O(n). That's the magic.

How to spot DP problems: "count the number of ways", "minimum/maximum cost", "is it possible" — these are your signals.<|end|>
<|user|>Explain Kadane's Algorithm for Maximum Subarray.<|end|>
<|meera|>Maximum Subarray: find the contiguous subarray with the largest sum.

Kadane's Algorithm is beautifully simple:
- Track current_sum: either extend the previous subarray or start fresh
- Track max_sum: best answer seen so far

```python
def max_subarray(nums):
    max_sum = curr_sum = nums[0]
    for num in nums[1:]:
        curr_sum = max(num, curr_sum + num)
        max_sum = max(max_sum, curr_sum)
    return max_sum
```

For [-2, 1, -3, 4, -1, 2, 1, -5, 4]:
- At 4: curr_sum resets to 4 (starting fresh is better)
- Subarray [4, -1, 2, 1] gives max sum = 6

Time: O(n), Space: O(1). This is the kind of elegant solution that makes you fall in love with algorithms.<|end|>
<|user|>What is a hash map and how does it work internally?<|end|>
<|meera|>A hash map (dictionary in Python, object in JS) stores key-value pairs with O(1) average lookup.

How it works internally:
1. Hash function converts key → integer (hash code)
2. Hash code % array_size = index (bucket)
3. Store the key-value pair at that index

Collision handling (when two keys map to same index):
- Chaining: each bucket is a linked list
- Open addressing: find next empty slot (linear probing, quadratic probing)

```python
# Python dict is a hash map
scores = {}
scores["alice"] = 95    # hash("alice") → some index
scores["bob"] = 87
print(scores["alice"])   # O(1) lookup
```

Average: O(1) for get/set/delete
Worst case: O(n) if everything collides (rare with good hash function)

Hash maps are everywhere — Two Sum, frequency counting, caching. Master this data structure.<|end|>
<|user|>Explain BFS and DFS with examples.<|end|>
<|meera|>Two fundamental graph/tree traversal strategies:

**BFS (Breadth-First Search)** — explore level by level. Uses a queue.
```python
from collections import deque
def bfs(graph, start):
    visited = {start}
    queue = deque([start])
    while queue:
        node = queue.popleft()
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
```
Use BFS for: shortest path (unweighted), level-order traversal.

**DFS (Depth-First Search)** — go as deep as possible, then backtrack. Uses a stack (or recursion).
```python
def dfs(graph, node, visited=set()):
    visited.add(node)
    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)
```
Use DFS for: cycle detection, topological sort, path finding, connected components.

Simple rule: need shortest path? BFS. Need to explore all paths? DFS.<|end|>
<|user|>What is the sliding window technique?<|end|>
<|meera|>Sliding window is a technique for problems involving contiguous subarrays/substrings. Instead of recalculating everything, you "slide" a window across the data.

Two types:
1. Fixed size: window size is given
2. Variable size: window expands/shrinks based on condition

Example — Longest substring without repeating characters:
```python
def longest_unique_substring(s):
    seen = set()
    left = result = 0
    for right in range(len(s)):
        while s[right] in seen:
            seen.remove(s[left])
            left += 1
        seen.add(s[right])
        result = max(result, right - left + 1)
    return result
```

The key insight: two pointers (left, right) define the window. Right expands, left shrinks when condition breaks.

Time: O(n) — each element is visited at most twice. Way better than O(n²) brute force.

Pattern signals: "contiguous subarray", "substring", "window of size k".<|end|>
<|user|>Explain merge sort and its time complexity.<|end|>
<|meera|>Merge sort is a divide-and-conquer sorting algorithm:
1. Split array in half
2. Recursively sort both halves
3. Merge the sorted halves

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

Time: O(n log n) — always, in every case. That's its superpower.
Space: O(n) — needs extra array for merging.

Compare: Quick sort is O(n log n) average but O(n²) worst case. Merge sort is consistent. Use it when stability matters.<|end|>
<|user|>What is a binary search tree?<|end|>
<|meera|>A Binary Search Tree (BST) is a binary tree where:
- Left child < parent
- Right child > parent
- This property holds for ALL nodes

```python
class TreeNode:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None

def insert(root, val):
    if not root:
        return TreeNode(val)
    if val < root.val:
        root.left = insert(root.left, val)
    else:
        root.right = insert(root.right, val)
    return root

def search(root, val):
    if not root or root.val == val:
        return root
    if val < root.val:
        return search(root.left, val)
    return search(root.right, val)
```

Average: O(log n) for search/insert/delete
Worst case (skewed tree): O(n) — basically a linked list

That's why balanced BSTs exist (AVL, Red-Black trees) — they guarantee O(log n).<|end|>
<|user|>Explain recursion with a simple example.<|end|>
<|meera|>Recursion is when a function calls itself to solve smaller versions of the same problem. Every recursive function needs:
1. Base case — when to stop
2. Recursive case — break problem into smaller pieces

```python
def factorial(n):
    if n <= 1:        # base case
        return 1
    return n * factorial(n - 1)  # recursive case
```

factorial(4) → 4 * factorial(3) → 4 * 3 * factorial(2) → 4 * 3 * 2 * 1 = 24

Think of it like Russian dolls — each doll opens to reveal a smaller one, until you hit the smallest.

Common pitfall: forgetting the base case = infinite recursion = stack overflow.

Where recursion shines: trees, graphs, divide-and-conquer, backtracking. If the problem has a recursive structure, use recursion.<|end|>

<|user|>What is time complexity and Big O notation?<|end|>
<|meera|>Time complexity measures how an algorithm's runtime grows as input size increases. Big O notation describes the upper bound (worst case).

Common complexities (fastest to slowest):
- O(1) — constant: hash map lookup
- O(log n) — logarithmic: binary search
- O(n) — linear: single loop
- O(n log n) — linearithmic: merge sort
- O(n²) — quadratic: nested loops
- O(2^n) — exponential: recursive fibonacci without memo
- O(n!) — factorial: generating all permutations

How to analyze:
```python
# O(n) — single loop
for i in range(n):
    print(i)

# O(n²) — nested loops
for i in range(n):
    for j in range(n):
        print(i, j)

# O(log n) — halving each step
while n > 0:
    n = n // 2
```

Rules:
1. Drop constants: O(2n) → O(n)
2. Drop lower terms: O(n² + n) → O(n²)
3. Worst case matters for Big O

When someone asks "optimize this", they mean reduce the Big O.<|end|>
<|user|>Explain the concept of greedy algorithms.<|end|>
<|meera|>Greedy algorithms make the locally optimal choice at each step, hoping it leads to a globally optimal solution.

Key property: greedy choice property — a local optimum leads to global optimum.

Classic example — Activity Selection:
```python
def max_activities(activities):
    # Sort by end time
    activities.sort(key=lambda x: x[1])
    result = [activities[0]]
    for i in range(1, len(activities)):
        if activities[i][0] >= result[-1][1]:
            result.append(activities[i])
    return result
```

Other greedy problems:
- Huffman coding
- Dijkstra's shortest path
- Fractional knapsack
- Minimum spanning tree (Kruskal's, Prim's)

Warning: greedy doesn't always work. 0/1 Knapsack? Greedy fails — you need DP.

How to know if greedy works: prove that the greedy choice is always safe, or just try it and see if it gives wrong answers on edge cases.<|end|>
<|user|>What is backtracking?<|end|>
<|meera|>Backtracking is a refined brute force — try all possibilities, but abandon a path as soon as you know it won't work.

Template:
```python
def backtrack(state, choices):
    if is_solution(state):
        result.append(state.copy())
        return
    for choice in choices:
        if is_valid(choice):
            state.append(choice)       # make choice
            backtrack(state, choices)   # explore
            state.pop()                # undo choice (backtrack)
```

Classic example — N-Queens:
Place N queens on an N×N board so no two attack each other.

```python
def solve_n_queens(n):
    result = []
    def backtrack(row, cols, diag1, diag2, board):
        if row == n:
            result.append(["".join(r) for r in board])
            return
        for col in range(n):
            if col in cols or (row-col) in diag1 or (row+col) in diag2:
                continue
            board[row][col] = 'Q'
            backtrack(row+1, cols|{col}, diag1|{row-col}, diag2|{row+col}, board)
            board[row][col] = '.'
    backtrack(0, set(), set(), set(), [['.']*n for _ in range(n)])
    return result
```

Backtracking problems: N-Queens, Sudoku solver, permutations, combinations, word search. The pattern is always the same — choose, explore, unchoose.<|end|>
<|user|>Explain heap data structure.<|end|>
<|meera|>A heap is a complete binary tree that satisfies the heap property:
- Min-heap: parent ≤ children (root is minimum)
- Max-heap: parent ≥ children (root is maximum)

Stored as an array where for index i:
- Parent: (i-1) // 2
- Left child: 2i + 1
- Right child: 2i + 2

```python
import heapq

# Min-heap in Python
heap = []
heapq.heappush(heap, 5)
heapq.heappush(heap, 2)
heapq.heappush(heap, 8)
print(heapq.heappop(heap))  # 2 (smallest)
```

Operations:
- Insert: O(log n)
- Extract min/max: O(log n)
- Peek: O(1)

Use cases:
- Priority queues
- Kth largest/smallest element
- Merge K sorted lists
- Dijkstra's algorithm
- Median finding (two heaps trick)

Interview tip: whenever you see "top K" or "Kth largest", think heap immediately.<|end|>
<|user|>What is a trie and when to use it?<|end|>
<|meera|>A trie (prefix tree) is a tree where each node represents a character. Paths from root to nodes form prefixes.

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word):
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True

    def search(self, word):
        node = self.root
        for ch in word:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return node.is_end

    def starts_with(self, prefix):
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return True
```

Time: O(m) where m = word length. Way faster than checking every word.

Use when: autocomplete, spell checker, word search, IP routing, prefix matching. If the problem involves prefixes, trie is your answer.<|end|>
